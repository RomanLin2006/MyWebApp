КОНСПЕКТ ПРОЕКТА: ВЕБ-ПРИЛОЖЕНИЕ ДЛЯ ВИЗУАЛЬНОГО МОНИТОРИНГА СТАТУСА АЛКОГОЛЬНЫХ ЛИЦЕНЗИЙ ПРЕДПРИЯТИЙ МОСКВЫ
===============================================================================================================

ТЕМА ПРОЕКТА
------------
Веб-приложение для визуального мониторинга статуса алкогольных лицензий предприятий Москвы

Статус: Тема согласована с преподавателем

ЦЕЛЕВЫЕ АУДИТОРИИ И ИХ ПОТРЕБНОСТИ
-----------------------------------
Приложение разрабатывается как универсальный инструмент для всех типов пользователей.
Один и тот же функционал полезен для:

1. Проверяющие органы (Роспотребнадзор, налоговая)
   - Быстрый мониторинг статуса лицензий
   - Контроль соблюдения законодательства
   - Планирование проверок

2. Собственники бизнеса
   - Отслеживание статуса своих лицензий
   - Мониторинг конкурентов
   - Анализ рынка

3. Инвесторы/арендодатели
   - Оценка надежности арендаторов
   - Проверка легальности деятельности
   - Анализ района для инвестиций

4. Потребители
   - Уверенность в легальности заведения
   - Информация о статусе лицензии
   - Поиск легальных заведений

ОСНОВНАЯ ИДЕЯ ПРИЛОЖЕНИЯ
------------------------
Универсальное веб-приложение с интерактивной картой Москвы для визуального мониторинга 
статуса алкогольных лицензий предприятий. Все функции доступны всем пользователям без 
разделения по ролям.

БАЗОВЫЙ ФУНКЦИОНАЛ (ОБЯЗАТЕЛЬНЫЙ)
---------------------------------
1. Интерактивная карта Москвы с маркерами предприятий
   - Отображение всех предприятий с алкогольными лицензиями на карте
   - Использование готовых геоданных из датасета (geoData, geodata_center)

2. Цветовая индикация статуса лицензии
   - ЗЕЛЁНЫЙ цвет - свежая лицензия (действующая, не скоро истекает)
   - ЖЁЛТЫЙ цвет - скорое истечение срока лицензии (требует внимания)
   - КРАСНЫЙ цвет - просроченная лицензия
   - Логика определения цветов на основе даты окончания срока действия лицензии

3. Детальная информация о предприятии
   - При клике на маркер/точку предприятия открывается карточка/модальное окно
   - Отображение всех доступных данных из датасета:
     * Наименование объекта и лицензиата
     * Адрес объекта лицензирования
     * Тип работ (РАО, РПО, РПА)
     * Реквизиты лицензии, номер в реестре ФСРАР
     * Даты начала и окончания действия лицензии
     * Текущее состояние лицензии
     * Контактная информация (email, ИНН, КПП)
     * Административный округ и район

4. Система фильтров для отбора предприятий
   - Фильтр по административному округу
   - Фильтр по району
   - Фильтр по типу работ (РАО, РПО, РПА)
   - Фильтр по статусу лицензии (действующая, истекающая, просроченная)
   - Фильтр по сроку действия лицензии
   - Поиск по наименованию объекта/лицензиата
   - Поиск по адресу
   - Поиск по ИНН/КПП
   - Фильтры применяются динамически к отображаемым на карте маркерам

5. Алгоритм k-means кластеризации
   - Географическая кластеризация предприятий по координатам
   - Анализ плотности распределения точек продажи алкоголя
   - Визуализация кластеров на карте (разные цвета/размеры для кластеров)
   - Возможность настройки количества кластеров (k)
   - Статистика по каждому кластеру (количество предприятий, распределение по типам работ, статусам)
   - Применение: выявление зон концентрации, анализ рынка, планирование проверок

6. Зоны риска с цветовой разметкой по плотности проблемных лицензий
   - Разделение карты Москвы на зоны (квадраты/полигоны) для анализа плотности
   - Расчёт для каждой зоны: количество предприятий с просроченными (красные) и истекающими (жёлтые) лицензиями
   - Цветовая заливка зон в зависимости от плотности проблемных лицензий:
     * ТЁМНО-КРАСНЫЙ - высокая плотность просроченных/истекающих лицензий (критическая зона)
     * ОРАНЖЕВЫЙ - средняя плотность проблемных лицензий (требует внимания)
     * ЖЁЛТЫЙ - низкая плотность проблемных лицензий (нормальная зона)
     * ЗЕЛЁНЫЙ - отсутствие или минимальное количество проблемных лицензий (благополучная зона)
   - Полупрозрачная заливка зон для сохранения видимости маркеров предприятий
   - При клике на зону показывается статистика: общее количество предприятий, количество проблемных, процент проблемных от общего числа
   - Настройка размера зон (гранулярность анализа) через интерфейс
   - Возможность фильтрации: показывать только зоны с определённым уровнем риска
   - Применение: быстрая визуальная оценка проблемных районов для проверяющих органов, анализ рисков для инвесторов, выявление зон с высокой концентрацией нарушений

7. Расширенный поиск и навигация
   - Поиск по адресу с автодополнением (живой поиск)
   - Поиск по названию предприятия с подсказками
   - Переход к предприятию по клику на результат поиска (центрирование карты)

8. Умные подсказки и анализ
   - При клике на предприятие показывать ближайшие аналогичные (по типу работ)
   - Подсветка предприятий одной сети (по ИНН лицензиата)
   - Индикатор "плотность конкурентов" - сколько похожих предприятий в радиусе N метров
   - Визуализация зон с высокой/низкой концентрацией предприятий

9. Информационные слои карты
   - Переключение между слоями: все предприятия / только РАО / только РПО / только РПА
   - Слой "проблемные зоны" - районы с высокой долей просроченных лицензий
   - Слой "активные зоны" - районы с высокой концентрацией действующих лицензий
   - Тепловая карта (heatmap) плотности предприятий


ИСТОЧНИК ДАННЫХ
---------------
Портал открытых данных Правительства Москвы
(Требуется указать конкретную ссылку на датасет в приложении и пояснительной записке)

АТРИБУТЫ ДАТАСЕТА
-----------------
Полный список полей, предоставляемых датасетом:

1. Наименование объекта лицензирования по уставным документам
2. global_id
3. Наименование объекта лицензирования
4. Административный округ
5. Район
6. Адрес объекта лицензирования
7. Наименование лицензиата
8. Юридический адрес лицензиата
9. Адрес электронной почты
10. ИНН лицензиата
11. КПП лицензиата
12. КПП обособленного подразделения
13. Тип работ (РАО, РПО, РПА)
   - РАО - Розничная продажа алкогольной продукции
   - РПО - Розничная продажа пива и пивных напитков
   - РПА - Розничная продажа алкогольной продукции в розлив
14. Реквизиты лицензии
15. Номер лицензии в сводном реестре ФСРАР
16. Дата начала срока действия лицензии
17. Дата окончания срока действия лицензии
18. Дата установки текущего состояния лицензии
19. Дата принятия решения
20. Текущее состояние лицензии
21. Наименование органа лицензирования
22. Уникальный номер адреса в государственном адресном реестре
   (формат: UUID, пример: f2d6a3b3-4f49-4a5c-91fd-9a433c3943280)
23. Кадастровый номер помещения
   (примечание: почти ни у кого нет, возможно не пригодится)
24. geoData
   (формат: JSON, пример: {coordinates=[37.947838, 55.703266], type=Point})
25. geodata_center
   (формат: JSON, пример: {coordinates=[37.947838, 55.703266], type=Point})


ЗАМЕТКИ ДЛЯ РАЗРАБОТКИ
----------------------
- Датасет содержит геоданные (geoData, geodata_center) - готовые координаты для карты
- Есть классификация по типу работ - можно использовать для разных типов маркеров
- Есть информация о сроках действия - основа для цветовой индикации
- Текущее состояние лицензии - важный атрибут для фильтрации
- Административное деление (округ/район) - удобно для фильтров и группировки
- Контактные данные (email, адреса) - можно использовать для связи/проверки
- ИНН/КПП - возможность поиска по налоговым реквизитам
- Уникальный номер адреса в ГАР - возможность интеграции с другими сервисами
- Номер лицензии в реестре ФСРАР - возможность прямой проверки через официальный реестр
