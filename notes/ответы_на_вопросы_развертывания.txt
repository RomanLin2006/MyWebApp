ОТВЕТЫ НА ВОПРОСЫ ПО РАЗВЕРТЫВАНИЮ И ИНСТРУМЕНТАМ
===================================================

ВОПРОС 1: Обязательно ли хранить приложение на сервере?
-------------------------------------------------------
ОТВЕТ: НЕТ, не обязательно использовать внешний хостинг!

По требованиям задания:
"Работа размещена на сервере [допускается использование локального]"

Это означает:
✓ Можно использовать ЛОКАЛЬНЫЙ сервер на вашем компьютере
✓ Можно использовать БЕСПЛАТНЫЙ внешний хостинг
✓ Главное - приложение должно РАБОТАТЬ и быть доступно для демонстрации

ВАРИАНТЫ:
1. Локальный сервер (самый простой вариант):
   - Запустить Flask на localhost:5000
   - Для демонстрации можно показать скриншоты или видео
   - Или показать работу напрямую на вашем компьютере

2. Локальный сервер с доступом из сети:
   - Использовать ngrok или подобные инструменты для временного доступа
   - Или настроить локальный сервер с доступом по локальной сети

3. Бесплатный хостинг (см. вопрос 2)

В КАКОМ ВИДЕ ПРЕДОСТАВИТЬ:
- Код в Git репозитории (обязательно!)
- Рабочее приложение (локально или на хостинге)
- Ссылка на репозиторий в пояснительной записке
- Скриншоты работы приложения
- Инструкция по запуску в README.md

ВОПРОС 2: Как и что хранить на сервере? Есть ли бесплатные хостинги?
---------------------------------------------------------------------
ОТВЕТ: Да, есть несколько бесплатных вариантов!

ЧТО ХРАНИТЬ:
1. Backend код (Python Flask)
2. Frontend код (HTML/CSS/JavaScript)
3. База данных MySQL (или подключение к внешней БД)

ВАРИАНТЫ БЕСПЛАТНОГО ХОСТИНГА:

1. PYTHONANYWHERE (РЕКОМЕНДУЕТСЯ для Python)
   - Сайт: https://www.pythonanywhere.com/
   - Бесплатный план: есть
   - Что включает:
     * Python 3.x
     * MySQL база данных (ограниченная)
     * Веб-сервер
     * SSL сертификат
   - Ограничения бесплатного плана:
     * 1 веб-приложение
     * Ограниченное время работы (может "засыпать")
     * Ограниченный трафик
   - Плюсы: очень просто для Python приложений
   - Минусы: может быть медленным

2. HEROKU (популярный, но ограничения)
   - Сайт: https://www.heroku.com/
   - Бесплатный план: был, но сейчас ограничен
   - Что включает:
     * Поддержка Python
     * PostgreSQL (бесплатно)
     * Автоматическое развертывание из Git
   - Ограничения: нужно добавить кредитную карту (но не списывает деньги)
   - Плюсы: очень удобное развертывание
   - Минусы: требует карту для регистрации

3. RENDER (хорошая альтернатива Heroku)
   - Сайт: https://render.com/
   - Бесплатный план: есть
   - Что включает:
     * Python поддержка
     * PostgreSQL (бесплатно)
     * Автоматическое развертывание из Git
   - Плюсы: современный, простой интерфейс
   - Минусы: приложения могут "засыпать" при неактивности

4. RAILWAY (современный вариант)
   - Сайт: https://railway.app/
   - Бесплатный план: есть (с ограничениями)
   - Что включает:
     * Поддержка Python
     * MySQL/PostgreSQL
     * Автоматическое развертывание
   - Плюсы: современный, хорошая документация

5. VERCEL / NETLIFY (только для frontend)
   - Подходят только для статических сайтов
   - Не подходят для Flask backend
   - Можно использовать для frontend, а backend на другом хостинге

РЕКОМЕНДАЦИЯ:
Для курсового проекта лучше всего:
1. PythonAnywhere - самый простой для Python приложений
2. Или локальный сервер - если не хотите возиться с хостингом

ЧТО ХРАНИТЬ НА СЕРВЕРЕ:
- Весь код приложения (backend + frontend)
- База данных MySQL (или подключение к ней)
- Файлы конфигурации

ЧТО НЕ ХРАНИТЬ:
- Секретные ключи и пароли (использовать переменные окружения)
- Большие файлы данных (хранить в БД или отдельном хранилище)

ВОПРОС 3: Можно ли использовать VS Code вместо IntelliJ IDEA?
---------------------------------------------------------------
ОТВЕТ: ДА, можно! И даже рекомендуется для frontend!

VS CODE - ЛУЧШЕ ДЛЯ:
✓ Frontend разработки (HTML/CSS/JavaScript)
✓ Работа с файлами проекта
✓ Легковесный и быстрый
✓ Отличные расширения для веб-разработки
✓ Встроенный терминал
✓ Git интеграция

INTELLIJ IDEA - ЛУЧШЕ ДЛЯ:
✓ Python backend разработки
✓ Работа с базами данных (встроенные инструменты)
✓ Рефакторинг кода
✓ Отладка Python приложений
✓ Работа с большими проектами

РЕКОМЕНДУЕМАЯ СХЕМА РАБОТЫ:
1. IntelliJ IDEA - для Python backend
   - Разработка Flask приложения
   - Работа с моделями БД
   - Отладка API

2. VS Code - для Frontend
   - Разработка HTML/CSS/JavaScript
   - Работа с Leaflet картами
   - Стилизация интерфейса

ИЛИ:
Можно использовать ТОЛЬКО VS Code для всего проекта:
- Установить расширение Python
- Установить расширение для работы с БД
- Установить расширения для HTML/CSS/JS

РАСШИРЕНИЯ VS CODE ДЛЯ РАБОТЫ:
1. Python (Microsoft) - для Python разработки
2. MySQL (cweijan) - для работы с БД
3. Live Server - для запуска frontend
4. Prettier - форматирование кода
5. ESLint - проверка JavaScript
6. GitLens - улучшенная работа с Git

ИТОГОВАЯ РЕКОМЕНДАЦИЯ:
----------------------
Для курсового проекта можно использовать ТОЛЬКО VS Code:
- Установить расширения Python, MySQL, Live Server
- Разрабатывать весь проект в одном редакторе
- Это проще и не требует переключения между программами

IntelliJ IDEA можно оставить как резервный вариант или не использовать вообще.

ПРАКТИЧЕСКИЙ СОВЕТ:
-------------------
Если вы уже знакомы с VS Code - используйте его!
Если больше знакомы с IntelliJ IDEA - используйте его для Python, VS Code для frontend.

Главное - выбрать один подход и придерживаться его на протяжении всего проекта.
